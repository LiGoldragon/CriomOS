## What landed

### Schema migration: nota-serde → nota-codec
- **nota-codec** ([`13a3eecaeadd`](https://github.com/LiGoldragon/nota-codec/commit/13a3eeca)
  + the codec-team's follow-ups): bool primitive + numeric / float / bytes,
  `Option<T>` accepts both explicit `None` ident and trailing-omission,
  `BTreeMap`/`HashMap` impls (`[(Entry k v) ...]` wire form),
  `read_string` accepts bare-ident, `NotaTryTransparent` derive.
- **horizon-rs** [`24740fbd`](https://github.com/LiGoldragon/horizon-rs/commit/24740fbd):
  full nota-serde → nota-codec sweep across `lib/src/{species,magnitude,name,pub_key,address,io,machine,proposal}.rs`,
  derive choices: `NotaEnum` on closed enums, `NotaTransparent` on
  String-newtypes, `NotaTryTransparent` on validating newtypes
  (preserves `try_new`), hand-rolled `NotaEncode`/`NotaDecode` for
  `YggAddress`/`NodeIp` (non-String inner types).
- **lojix-cli** [`454f2cdc`](https://github.com/LiGoldragon/lojix-cli/commit/454f2cdc):
  dep swap, `Decoder::nota` parse path, vendor hashes refreshed.
- **goldragon datom** [`b25950fc`](https://github.com/LiGoldragon/goldragon/commit/b25950fc):
  v2→v3 rewrite (`<…>` → `[…]`, map-entry shorthand → `(Entry "k" v)`,
  bare-ident strings where allowed). Subsequent commits also added
  prom's full pubkeys + per-node `online` + `nb_of_build_cores`.

### NodeProposal schema additions
- horizon-rs [`24740fbd`](https://github.com/LiGoldragon/horizon-rs/commit/24740fbd)

## Quick-reference: deploying the new criomos

```bash
# from ouranos:
lojix-cli deploy --cluster goldragon --node <node> \
  --source /home/li/git/goldragon/datom.nota \
  --action <test|boot|switch> \
  --criomos github:LiGoldragon/CriomOS/<rev>   # pin rev to bypass eval cache
```

`--action test` = activate live, no boot-loader change (rebooting reverts).
`--action boot` = update boot loader only, takes effect next boot.
`--action switch` = both.

For `--action test` on the new criomos, the dbus → dbus-broker switch
inhibitor blocks live activation; use `boot` and reboot.
